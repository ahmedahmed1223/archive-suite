# مهام Masar الحالية

> هذا هو سجل **المهام المفتوحة فقط** للمسار القانوني Laravel + Next.js. تفاصيل كل بند مكتمل مؤرشفة في [`ChangeLog.md`](ChangeLog.md) ولا تعود هنا إلا بمتابعة جديدة ذات معرّف مستقل.

> **للوكلاء المنفذين:** تبدأ كل ميزة بـTDD، وتبقى في ملفاتها المملوكة. تُنفذ المميزات المتقاربة في دفعة واحدة، ثم تُراجع وتُفحص الدفعة قبل الدمج؛ ولكل ميزة مقبولة commit مستقل.

> **قاعدة إعادة الفتح:** لا يعود أي بند مكتمل إلى هذا السجل إلا بمتابعة جديدة ذات معرّف مستقل، أو بعد توفر شرط خارجي موثق يحوله إلى عمل محلي قابل للتحقق.

## الحالة

- الحالة العامة **NO-GO**: أُغلقت مانعات P0 الأصلية، لكن لا يزال الإصدار محجوبًا بالمهام المفتوحة، القبول الحي، وبوابات RC/GA.
- يوجد 46 بند V1 مفتوح و4 تحققات اختيارية معطلة افتراضيًا؛ بوابة V1-406 تتولى منع الإصدار عند بقاء أي مانع مفتوح.
- التصميم الحاكم: [`docs/superpowers/specs/2026-07-14-daily-use-dual-runtime-design.md`](docs/superpowers/specs/2026-07-14-daily-use-dual-runtime-design.md).
- المسار القانوني للعمل الجديد هو `archive-next/` و`archive-laravel/`؛ أعمال التثبيت والتشغيل في `scripts/` و`infra/`.

## بوابات التنفيذ لكل مهمة مفتوحة

- ابدأ باختبار فاشل يثبت السلوك أو العقد، ثم أقل تنفيذ ينجحه، ثم refactor محدود.
- تحقق بأضيق اختبار أو typecheck/Laravel/Node متأثر أثناء العمل. بعد استقرار الدفعة شغّل `pnpm typecheck` و`pnpm test:next` أو `pnpm verify:laravel`، و`pnpm verify:api-contracts` عند لمس العقد، وbuild واحد للدفعة.
- دفعة Setup/Native تشغّل أيضًا `node --test scripts/control-center.test.mjs scripts/platform-contract.test.mjs` و`pnpm verify:infra`.
- دفعة الرحلات والأدوار والوصولية تشغّل `pnpm verify:laravel-next:live` وتحفظ screenshots/traces/reports.
- مهام clean-host لا تُغلق بمحاكاة محلية؛ يجب إرفاق platform/version/mode/source/manifest ونتيجة كل scenario ID من الجهاز المستهدف.
- بعد القبول: commit مستقل، ثم إضافة سجل موجز أو موسع في `ChangeLog.md` وإزالة البند من هذا الملف.
