# Offline-first Sync — Local Mirror + Write Queue + Conflict Resolution

> الحالة: مواصفة مقبولة لـ Sub-project 3 بعد M1-M4. التنفيذ اللاحق يجب أن يبقى خلف منافذ `@archive/core` ولا يغيّر صفحات المنتج مباشرة.

## الهدف

جعل النسخة المحلية مرآة قابلة للعمل دون اتصال لخادم SQL. يقرأ المستخدم آخر حالة معروفة من IndexedDB/SQLite المحلي، وتدخل الكتابات في طابور محلي عند انقطاع الشبكة، ثم تُزامَن عند عودة الاتصال مع حل تعارض واضح.

## النطاق

- مرآة محلية لكل الكيانات التي تدخل snapshot الحالي: المواد، الأنواع، القاموس، الوسوم، المجموعات، المشاريع، التعليقات، الإشعارات، الإعدادات غير السرية.
- طابور كتابة محلي append-only يحتوي: `id`, `entity`, `op`, `payload`, `baseVersion`, `createdAt`, `actorId`, `idempotencyKey`.
- مزامنة delta عبر الخادم: `sinceCursor` للقراءة، ودفعات writes للرفع.
- واجهة حالة يومية: online, degraded, offline, syncing, conflicts.

## خارج النطاق

- نقل معمارية `CLOUD-MediaDB` أو Firestore.
- تخزين أسرار الخادم داخل snapshot المحلي.
- حل تعارض صامت للحقول عالية الخطورة مثل الصلاحيات أو إعدادات الخادم.

## تدفق القراءة

1. عند بدء التطبيق، تُحمّل المرآة المحلية فوراً.
2. إذا كان الخادم متاحاً، يطلب العميل delta منذ آخر cursor.
3. تُطبّق delta محلياً بترتيب server revision.
4. تتحول حالة الاتصال إلى online أو degraded حسب نتيجة `/api/health`.

## تدفق الكتابة

1. كل عملية حفظ تُكتب محلياً أولاً مع optimistic UI.
2. إذا كان الاتصال online، تُرسل العملية فوراً مع `idempotencyKey`.
3. إذا فشل الاتصال، تبقى العملية pending في الطابور وتظهر شارة "بانتظار المزامنة".
4. عند العودة، تُرسل الدفعات بحد أقصى قابل للضبط، مع backoff عند الفشل.

## التعارضات

- إذا تطابق `baseVersion` مع server version: يطبّق الخادم العملية ويعيد revision جديداً.
- إذا تغيّر نفس الحقل على الخادم والعميل: ينتج conflict record.
- الحقول النصية الطويلة تعرض مقارنة "نسخة الخادم / نسختك / دمج يدوي".
- القوائم مثل الوسوم تستخدم merge آمن بإزالة التكرار عند عدم وجود حذف متعارض.
- الصلاحيات والإعدادات الحساسة تحتاج قرار مدير.

## واجهة المستخدم

- في شريط الحالة: "متصل"، "يعمل دون اتصال"، "تتم المزامنة"، "توجد تعارضات".
- في مركز البيانات: عدد عمليات الطابور، آخر cursor، آخر خطأ، زر إعادة المحاولة، وزر تصدير طابور الدعم.
- في صفحة التعارضات: فلترة حسب الكيان، معاينة أثر، اختيار نسخة الخادم أو النسخة المحلية أو دمج يدوي.

## اختبارات القبول

- قطع الاتصال ثم إنشاء مادة لا ينشئ نسخاً مكررة بعد عودة الاتصال.
- نقر الحفظ خمس مرات ينتج عملية واحدة بسبب `idempotencyKey`.
- تعديل نفس عنوان المادة من جهازين ينتج conflict واضحاً لا overwrite صامت.
- delta بعد restart لا يعيد تحميل snapshot كامل إذا كان cursor صالحاً.
- لا تحتوي ملفات النقل أو الطابور على كلمات مرور أو مفاتيح FileStore أو AI.
