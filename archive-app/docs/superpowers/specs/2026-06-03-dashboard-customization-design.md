# مواصفات: تخصيص لوحة التحكم بشبكة حرة (Dashboard Customization)

**التاريخ:** 2026-06-03
**النطاق:** ميزة جديدة — شبكة حرة قابلة للسحب/التحجيم لعناصر لوحة التحكم، مع حفظ التخطيط لكل مستخدم.

## 1. القرارات

| القرار | الاختيار |
|--------|---------|
| نموذج التخصيص | شبكة حرة كاملة (drag أينما + resize بمقبض) |
| المحرّك | `react-grid-layout` (ResponsiveGridLayout) |
| الأعمدة | 12 على سطح المكتب · ينهار لعمود واحد على الجوال |
| التحرير | على سطح المكتب فقط؛ الجوال عرض-فقط للترتيب المحفوظ |
| الحفظ | لكل مستخدم في `settings.ui.dashboardLayout` |
| الارتفاع | **تلقائي افتراضياً** (يتبع المحتوى) مع إمكانية تثبيت يدوي |

## 2. اللوحات (Panel Registry)

تسعة عناصر، كل واحد بمعرّف ثابت:

| id | العنوان | ملاحظة |
|----|--------|--------|
| `hero` | مركز التحكم | الهيرو (بحث + حالة) |
| `reportStrip` | المؤشرات | شريط البطاقات الأربع |
| `dailyFocus` | أولويات اليوم | يظهر عند وجود عناصر |
| `operations` | إجراءات العمليات | |
| `savedViews` | عروض محفوظة | يظهر عند وجود عروض |
| `distribution` | توزيع المحتوى | |
| `orgMetrics` | مؤشرات تنظيمية | |
| `recentItems` | آخر المواد | |
| `recentActivity` | آخر نشاط | |

سجلّ مركزي `DASHBOARD_PANELS` يربط `id → { title, defaultLayout, render }`. هذا يفصل **محتوى اللوحة** عن **موضعها في الشبكة**.

## 3. نموذج البيانات

```ts
// settings.ui.dashboardLayout
{
  version: 1,
  items: {
    [panelId]: { x, y, w, h, hidden?: boolean, autoHeight?: boolean }
  }
}
```

- `autoHeight: true` (افتراضي): الارتفاع يتبع المحتوى — يُقاس عبر `ResizeObserver` ويُحوّل لعدد صفوف الشبكة (`h`)؛ مقبض الارتفاع معطّل (عرض فقط).
- `autoHeight: false`: المستخدم ثبّت ارتفاعاً يدوياً؛ يبقى كما هو.
- `hidden: true`: اللوحة خارج الشبكة، متاحة لإعادة الإضافة.

## 4. الوحدات (ملفات مركزة)

| ملف | المسؤولية |
|-----|-----------|
| `src/features/dashboard/dashboardLayoutModel.js` | دوال نقية: `getDefaultDashboardLayout()` · `normalizeDashboardLayout(stored, availableIds)` (تطبيع + دمج الجديد + إسقاط المحذوف + رجوع للافتراضي عند التلف) · `toGridLayout(layout, breakpoint)` · `applyGridChange(layout, rglLayout)` · `hasDashboardLayoutDraftChanges` (موجود — يُعاد استخدامه) |
| `src/features/dashboard/DashboardGrid.jsx` | غلاف `ResponsiveGridLayout` + منطق وضع التحرير + ResizeObserver للارتفاع التلقائي |
| `src/features/dashboard/dashboardPanels.js` | سجلّ اللوحات `DASHBOARD_PANELS` (id → meta) |
| `src/pages/DashboardPage.jsx` | يبني محتوى كل لوحة ويمرّره للشبكة (تقليص الـ render الحالي) |
| `src/styles/v4-identity.css` | تنسيقات الشبكة/المقابض/الإطارات المنقّطة (scoped لـ v4 وغيره) |

## 5. وضع التحرير (Edit Mode) + محسّنات UX

- زر **«تخصيص اللوحة»** أعلى اللوحة → يدخل وضع التحرير.
- **مقبض السحب في رأس اللوحة فقط** (`draggableHandle=".va-panel-drag"`) حتى لا يتعارض مع محتوى اللوحة.
- مقابض التحجيم تظهر فقط في وضع التحرير؛ في الارتفاع التلقائي يُتاح تحجيم **العرض فقط**.
- **شريط أدوات تحرير ثابت:** حفظ · إلغاء (يرجع للمحفوظ) · استعادة الافتراضي (بتأكيد عبر `appConfirm`).
- **قائمة إظهار/إخفاء** اللوحات (لوحة مخفية تظهر في القائمة لإعادة إضافتها).
- **حارس تغييرات غير محفوظة:** عند محاولة الخروج/التنقّل مع وجود مسودّة غير محفوظة → `appConfirm`.
- **مفتاح الارتفاع التلقائي** لكل لوحة (تبديل تلقائي/يدوي) في رأسها أثناء التحرير.
- `compactType: "vertical"` لإزالة الفجوات تلقائياً.
- **إتاحة لوحة المفاتيح:** كل أزرار التحرير (تخصيص/حفظ/إلغاء/استعادة/إظهار-إخفاء/تبديل الارتفاع) أزرار حقيقية قابلة للتركيز؛ مقابض RGL تبقى للفأرة/اللمس مع `aria-label`.
- **احترام الحركة المخفّضة:** `prefers-reduced-motion` → تعطيل انتقالات RGL.
- **توست تأكيد** عند الحفظ ("تم حفظ تخطيط اللوحة").

## 6. الاستجابة (Responsive)

- breakpoints: `lg` (≥1024, 12 عمود) · `md` (≥768, 8 أعمدة) · `xs` (<768, عمود واحد، عرض-فقط، زر التخصيص مخفي).
- على الجوال: تُعرض اللوحات مكدّسة بترتيب `y` المحفوظ.

## 7. معالجة الأخطاء

- تخطيط مخزَّن تالف أو نسخة غير معروفة → `getDefaultDashboardLayout()`.
- لوحة جديدة أُضيفت لاحقاً وغير موجودة في المخزَّن → تُضاف بموضعها الافتراضي (أسفل الشبكة).
- لوحة مُزالت من الكود لكنها في المخزَّن → تُتجاهَل.
- لوحة شرطية بلا محتوى (مثل `savedViews` بلا عروض) → لا تُرسَم، ومكانها يُطوى تلقائياً (compact).

## 8. الاختبار

- **node-assert** (`verify-modules`): `getDefaultDashboardLayout` يعيد كل المعرّفات · `normalizeDashboardLayout` (دمج جديد/إسقاط محذوف/رجوع عند التلف) · `toGridLayout`/`applyGridChange` · `hasDashboardLayoutDraftChanges`.
- **a11y matrix:** صفحة dashboard تبقى خالية من مخالفات serious + تباين v4.
- **build:** `npm run check` + `build:cloud` ينجحان (تأكيد تحزيم react-grid-layout في الملف الواحد).
- **بصري:** تحقق في المعاينة (دخول/خروج وضع التحرير، سحب، تحجيم، إظهار/إخفاء، حفظ، إعادة تحميل = ثبات التخطيط).

## 9. مراحل التنفيذ

1. تثبيت `react-grid-layout` + استيراد CSS الخاص بها.
2. `dashboardLayoutModel.js` + اختباراته (TDD).
3. `dashboardPanels.js` (سجلّ اللوحات) + استخراج محتوى اللوحات من DashboardPage.
4. `DashboardGrid.jsx` (الشبكة + وضع التحرير + الارتفاع التلقائي).
5. ربط الحفظ بـ `settings.ui.dashboardLayout` + شريط الأدوات + الحارس.
6. تنسيقات الشبكة/المقابض (RTL + reduced-motion).
7. الاستجابة + إخفاء التحرير على الجوال.
8. البوابات + التحقق البصري.
