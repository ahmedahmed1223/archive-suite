# مسار v1.0.0-rc.1

أول مرشح إصدار لنظام إدارة الأرشيف الموزع. منصة آمنة وقابلة للتشغيل ذاتياً لحفظ الوسائط وتنظيمها ومشاركتها عبر HTTP/HTTPS على الخوادم المحلية أو السحابية.

## أبرز ما في الإصدار

- **أمن وتحكم وصول:** رفض كلمات المرور الافتراضية (V1-101)، نافذة RBAC كاملة لكل نقطة توصيل (V1-102)، حماية refresh cookie بـ Origin/CSRF (V1-103)
- **نسخ احتياطي واستعادة كاملة:** كل الجداول والملفات مع checksums (V1-121)، استعادة آمنة غير مدمّرة مع فحص السلامة (V1-122)
- **موثوقية الوظائف:** timeouts/retries/idempotency/cancel حقيقي مع تنقية الأخطاء (V1-113)
- **التحقق من الرفع:** فحص MIME/magic bytes وقاعدة quarantine وأسماء UUID آمنة (V1-112)
- **اختبار آمن للترقيات:** أمر `archive:migrate-safe` مع preflight وbackup تلقائي (V1-203)
- **صور Dockerfile ثابتة:** Node 22.13.0 و PHP 8.4.23 و Composer 2.9.5 معقودة بـsha256 (V1-204)
- **سلسلة توريد موقعة:** SBOM وchecksum وتوقيع Cosign keyless وتحقق GitHub OIDC (V1-205)
- **حزمة Docker air-gapped:** بدون اتصال بالشبكة، مثبتات Windows/Linux، فحص manifest كامل (V1-206)

## الأمن

- **رفض المدير الافتراضي الضعيف:** كلمات المرور الخاصة بـ admin و تبديل secure_cookies يُحقّق أثناء التمهيد (V1-101)
- **نافذة أدوار:** admin/editor/viewer محمي على كل endpoint مع اختبارات Gate (V1-102)؛ وحدات media معزولة بالملكية
- **حماية Refresh token:** مسار كوكي مقيد بـ `/api/v1/auth/refresh`، فحص Origin صريح، throttle 30/دقيقة (V1-103)
- **أسرار المشاركة:** بدل query strings بـ X-Share-Password في body/header (V1-104)
- **الاحتواء والملكية:** media_jobs تفحص created_by وMediaPathGuard يمنع traversal (V1-111)
- **التحقق من الرفع:** finfo sniffing بعد الامتداد، UUID للملفات، حجر صحي قبل التخزين (V1-112)
- **موثوقية الوظائف:** 900s timeout، 3 retries مع backoff 30/120/300s، إلغاء حقيقي مع فحص cancelled قبل المعالجة (V1-113)

## التشغيل والنشر

- **nginx + PHP-FPM إنتاجي:** استبدل PHP dev server، healthchecks عميقة (DB/Redis/storage)، صلاحيات `www-data` (V1-202)
- **ترقيات آمنة:** preflight + backup تلقائي + maintenance عند الحاجة فقط + معاملة معزولة مع rollback على الفشل (V1-203)
- **صور immutable:** Node وPHP وComposer وruntime (pgvector/Redis/Caddy) معقودة بـ `version@sha256` في Compose (V1-204)
- **supply chain:** SBOM/provenance/digest manifest/checksums SHA256SUMS/Cosign signature + GitHub OIDC verification (V1-205)
- **حزمة air-gapped:** 5 صور Docker + manifest + tar.gz موقّع، بدون registry pull، مثبتات PowerShell/bash (V1-206)

## التجربة

- **wizard تفاعلي:** `setup.bat` / `setup.sh` يوجّه الخادم والحساب والإعدادات على الويب (يدمج V1-004)
- **أكواد خطأ مستقرة:** 8 رموز API ثابتة مع renderer مركزي وتنقية 500 في الإنتاج (error-codes من V1-002)
- **حفظ البيانات الآلي والحذف:** جلسات/سجل/وظائف/نسخ احتياطية محذوفة حسب retention windows (V1-123)
- **RPO/RTO قابلة للقياس:** `backup:dr-drill` يقيس RTO الفعلي، RPO من filemtime آخر نسخة (V1-123)

## معروف ومؤجل

- **V1-207 observability:** correlation E2E معلقة على canonical Docker execution (اختبارات محلية نجحت)
- **كلمات مرور السحابة:** pg_dump-level وsnapshot المزود غير مطبق (يتطلب تصميم الرابط وIntegrations)
- **حصص التخزين:** حد فردي 600MB فقط موجود؛ حصص لكل user/org مؤجلة
- **RC ميداني:** اختبار حي في 3-5 بيئات قبل GA قادم
- **ODBC + collections + tags:** قائمة طويلة من non-V1-1.0 scope (موثّق في TASKS.md)
