# التصميم المتجاوب للتنقل ولوحات الواجهة

## الهدف

جعل التنقل الرئيسي والإشعارات والمساعدة قابلة للاستخدام على الهاتف واللوحي وسطح المكتب، مع اتجاه RTL صحيح وزر تنقل ظاهر دائمًا.

## القرار

- من `1024px` فأعلى: قائمة جانبية ثابتة بجانب المحتوى، يمكن طيّها إلى أيقونات مع حفظ الاختيار محليًا.
- من `768px` إلى `1023px`: قائمة جانبية مضغوطة قابلة للتوسيع.
- دون `768px`: Drawer فوق المحتوى يفتحه زر واضح في الرأس، ويغلق بزر الإغلاق أو بالخلفية أو `Escape`، ثم يعيد التركيز إلى زر الفتح.
- تبقى الإشعارات منسدلة على الشاشات الواسعة وتتحول إلى Drawer بارتفاع آمن للشاشة على الهاتف؛ الرأس والأدوات ثابتة والقائمة قابلة للتمرير.
- تستخدم صفحة المساعدة عرضًا مرنًا وقيود ارتفاع ومسافات متجاوبة، من دون اقتطاع أو تجاوز للنصوص الطويلة.

## حدود التنفيذ

- لا تغيير في المسارات أو الصلاحيات أو API الإشعارات.
- تستخدم التغييرات tokens وطبقات CSS الحالية فقط.
- حد لمس لا يقل عن 44px على الهاتف، والحركة تحترم `prefers-reduced-motion`.

## القبول

1. زر تنقل مسمّى وواضح يحدّث `aria-expanded` و`aria-controls`.
2. تعمل القائمة بالماوس واللمس ولوحة المفاتيح، وتعود البؤرة إلى الزر عند الإغلاق.
3. لا يتجاوز التنقل أو الإشعارات أو المساعدة الشاشة عند 375 و768 و1280px.
4. تظل قائمة الإشعارات قابلة للتمرير دون إخفاء العنوان أو الإجراءات.
